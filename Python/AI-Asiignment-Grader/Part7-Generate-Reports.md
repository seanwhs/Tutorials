# Part 7 — Generating Professional PDF Grading Reports

Up to this point, Markly has evolved into a capable AI grading assistant.

A teacher can:

* Upload a PDF assignment
* Upload a Microsoft Word document
* Upload a scanned worksheet
* Upload a handwritten assignment
* Select a subject
* Receive AI-generated feedback in the browser

That already makes the system useful, but there is a structural limitation.

Once the session ends, the feedback disappears.

In real classroom workflows, grading output is not ephemeral. Teachers need to:

* Print grading reports
* Email feedback to students
* Archive assessments for records
* Upload results to an LMS
* Maintain long-term grading history

Copying results manually into Word or Google Docs is not sustainable. So we automate this step.

By the end of this chapter, Markly will generate **downloadable, professional PDF grading reports** automatically.

---

# Why PDFs?

PDF (Portable Document Format) is the standard for document exchange because it preserves layout across environments.

Compared to editable documents:

* Layout remains consistent across devices
* Fonts and spacing are preserved
* Files are print-ready by default
* Content cannot be accidentally modified
* Ideal for formal assessment records

This makes PDF the correct format for grading artifacts.

A typical report will look like:

```text
--------------------------------------------------

             MARKLY GRADING REPORT

Student:
John Tan

Subject:
Mathematics

Assignment:
Algebra Worksheet

----------------------------------------------

Strengths
• Strong understanding of algebraic manipulation
• Clear working shown

Areas for Improvement
• Sign errors when moving terms
• Careless arithmetic in steps

Suggestions
• Practice negative number operations

Final Grade
8 / 10

Generated by Markly

--------------------------------------------------
```

# Introducing Report Generation with ReportLab

We will use the Python library **ReportLab**.

ReportLab allows us to generate PDFs programmatically rather than using a visual editor.

Instead of designing documents manually, we define them in Python code.

Conceptually:

```text
Start with blank page
→ Add title
→ Add metadata
→ Add feedback
→ Export PDF
```

ReportLab provides two main levels:

* `canvas` (low-level, coordinate-based drawing)
* `Platypus` (high-level document framework)

We will use both to understand the progression, but settle on Platypus for real use.

# Why Not Keep This in app.py?

As features grow, UI files tend to accumulate unrelated logic:

```text
app.py → grading + file handling + AI + PDF + UI
```

This quickly becomes unmaintainable.

We enforce a clean separation:

| Module      | Responsibility   |
| ----------- | ---------------- |
| app.py      | UI (Panel)       |
| engine.py   | AI interaction   |
| personas.py | grading behavior |
| utils.py    | file processing  |
| report.py   | PDF generation   |

This keeps each layer predictable and testable.

# Step 1 — Minimal PDF Generation (Canvas)

Start with a simple test using ReportLab’s canvas API.

```python
import io
from reportlab.pdfgen import canvas


def create_pdf():
    buffer = io.BytesIO()
    pdf = canvas.Canvas(buffer)

    pdf.drawString(100, 750, "Hello from Markly!")

    pdf.save()
    buffer.seek(0)

    return buffer
```

---

## Understanding BytesIO

Instead of writing to disk, we write into memory.

Flow:

```text
ReportLab → writes PDF → BytesIO → returned to app → downloaded
```

This avoids filesystem dependency and improves deployment portability.

---

## Understanding Coordinates

PDF pages use a coordinate system:

* Origin (0,0) is bottom-left
* Y increases upward

```text
(0,800)
+----------------------+
|                      |
|   text here          |
|                      |
+----------------------+
(0,0)
```

Example:

```python
pdf.drawString(100, 750, "Hello")
```

```python
    f.write(pdf.read())
```

# Step 2 — Structured Report (Canvas Version)

Now we expand the output:

```python
import io
from reportlab.pdfgen import canvas


def create_pdf_report(student, subject, filename, feedback):
    buffer = io.BytesIO()
    pdf = canvas.Canvas(buffer)

    y = 780

    pdf.setFont("Helvetica-Bold", 18)
    pdf.drawString(72, y, "Markly Grading Report")

    y -= 40

    pdf.setFont("Helvetica", 11)
    pdf.drawString(72, y, f"Student: {student}")
    y -= 20

    pdf.drawString(72, y, f"Subject: {subject}")
    y -= 20

    pdf.drawString(72, y, f"Assignment: {filename}")

    pdf.save()
    buffer.seek(0)

    return buffer
```

---

## Limitation of Canvas Approach

This breaks down when handling real AI feedback:

```text
Long paragraph → becomes one long line → spills off page
```

Example issue:

```python
pdf.drawString(72, y, feedback)
```

No wrapping, no layout control.

We need a higher-level abstraction.

# Step 3 — Switching to Platypus (Recommended)

We now use **ReportLab Platypus**, which provides document-style layout.

Instead of manual positioning, we define a document structure.

---

## Platypus-Based Implementation

```python
import io
from reportlab.lib.styles import getSampleStyleSheet
from reportlab.platypus import SimpleDocTemplate, Paragraph, Spacer


def create_pdf_report(student, subject, filename, feedback):

    buffer = io.BytesIO()
    document = SimpleDocTemplate(buffer)

    styles = getSampleStyleSheet()
    story = []

    # Title
    story.append(Paragraph(
        "<b>Markly Grading Report</b>",
        styles["Title"]
    ))

    story.append(Spacer(1, 12))

    # Metadata
    story.append(Paragraph(f"<b>Student:</b> {student}", styles["BodyText"]))
    story.append(Paragraph(f"<b>Subject:</b> {subject}", styles["BodyText"]))
    story.append(Paragraph(f"<b>Assignment:</b> {filename}", styles["BodyText"]))

    story.append(Spacer(1, 20))

    # Feedback
    story.append(Paragraph(
        feedback.replace("\n", "<br/>"),
        styles["BodyText"]
    ))

    document.build(story)
    buffer.seek(0)

    return buffer
```

---

## Why Platypus is Better

It automatically handles:

* Text wrapping
* Page breaks
* Spacing
* Paragraph formatting
* Multi-page documents

This is significantly more stable for real grading output.
