# Part 14: AI Summary with the Vercel AI SDK (Free Model Selector)

## Concept

This is the "wow factor" feature: a plain-English summary like "DBS dropped 2.3% this week. Earnings beat, but rate cut fears. Analyst sentiment: Bullish. Dividend yield now 5.8%." We build this with the Vercel AI SDK's `generateText`/`streamText` functions, and crucially, a model-selector abstraction so the app only ever calls free-tier models, and swapping between them is a one-line config change rather than a rewrite.

## Step 1: Free model providers we support

- **Groq**: free tier, extremely fast inference, hosts open models like Llama 3.1 8B/70B versions at no cost within generous rate limits. Package: `@ai-sdk/groq`.
- **Google Gemini via Google AI Studio**: free tier includes Gemini Flash-tier models with generous daily free quota. Package: `@ai-sdk/google`.
- **OpenRouter**: aggregates many providers and explicitly lists a set of ":free" suffixed models (e.g., community Llama/Mistral endpoints) at zero cost, useful as a third fallback option. Access via the OpenAI-compatible provider pointed at OpenRouter's base URL.

We implement all three behind one interface and let the developer pick a default, with automatic fallback if a call fails (mirroring the pattern from Part 5's market-data fallback).

## Step 2: Dependencies

Part 2 already installed `ai`, `@ai-sdk/google`, `@ai-sdk/groq`, and `@ai-sdk/openai` (the last one is required for the OpenRouter option, since OpenRouter is accessed via an OpenAI-compatible endpoint). If you skipped Part 2's full install list, run:

## Step 3: The model registry

Create `src/lib/ai/models.ts`:

```typescript
// src/lib/ai/models.ts
import { createGroq } from "@ai-sdk/groq";
import { createGoogleGenerativeAI } from "@ai-sdk/google";
import { createOpenAI } from "@ai-sdk/openai";

const groq = createGroq({ apiKey: process.env.GROQ_API_KEY });
const google = createGoogleGenerativeAI({ apiKey: process.env.GOOGLE_GENERATIVE_AI_API_KEY });
const openrouter = createOpenAI({
  apiKey: process.env.OPENROUTER_API_KEY,
  baseURL: "https://openrouter.ai/api/v1",
});

export type FreeModelId = "groq-llama" | "gemini-flash" | "openrouter-free";

export const FREE_MODELS: Record<FreeModelId, { label: string; model: any }> = {
  "groq-llama": {
    label: "Groq — Llama 3.1 8B Instant (free)",
    model: groq("llama-3.1-8b-instant"),
  },
  "gemini-flash": {
    label: "Google Gemini Flash (free tier)",
    model: google("gemini-2.0-flash"),
  },
  "openrouter-free": {
    label: "OpenRouter — Community Free Model",
    model: openrouter("meta-llama/llama-3.1-8b-instruct:free"),
  },
};

export const DEFAULT_MODEL_ID: FreeModelId = "groq-llama";

export function getModel(id: FreeModelId = DEFAULT_MODEL_ID) {
  return FREE_MODELS[id]?.model ?? FREE_MODELS[DEFAULT_MODEL_ID].model;
}
```

Add `GROQ_API_KEY`, `GOOGLE_GENERATIVE_AI_API_KEY`, and optionally `OPENROUTER_API_KEY` to `.env.local`. Only Groq and Gemini are strictly required; OpenRouter is an optional third option.

> **Model ID freshness note:** Free-tier model IDs rotate over time — providers retire model versions and add new ones. If a `generateText` call fails with a "model not found"-style error, check the provider's current model list (console.groq.com/docs/models, ai.google.dev/models, openrouter.ai/models with the "free" filter) and update the ID strings above. Centralizing model IDs in this one file means a model retirement is a one-line fix here, not a hunt through the codebase.

## Step 4: Fallback-aware generation helper

Create `src/lib/ai/summarize.ts` implementing a `generateWithFallback` function that tries models in a preferred order (e.g., the user's selected model first, then the other two as automatic fallbacks) using the AI SDK's `generateText`, catching errors (rate limits, transient provider outages) and moving to the next model, finally throwing a combined error if all three fail. Log which model actually answered so we can display "Generated by: Groq Llama 3.1" in the UI for transparency.

## Step 5: Building the prompt

Create `src/lib/ai/prompts.ts` with a `buildStockSummaryPrompt` function that assembles a compact, structured prompt from: ticker, company name, current price, change percent over 1 week, P/E ratio, dividend yield, RSI reading and its interpretation label from Part 10, and up to 3 recent news headlines (stubbed as an empty array until Part 17 wires real news in). The prompt explicitly instructs the model to: write 2-4 sentences, plain English suitable for a retail investor, mention the price move and a plausible driver if inferable from the given data, state a sentiment label (Bullish/Bearish/Neutral) based only on the provided figures, and avoid making up facts not present in the input data. This last instruction matters: we are constraining the model to summarize given structured data, not hallucinate news it wasn't given.

## Step 6: The AI summary API route

> **Next.js 16 note:** As with every dynamic route in this series (see Part 7), `params` here is `Promise`-based and must be awaited.

Create `src/app/api/stocks/[ticker]/ai-summary/route.ts`:

```typescript
// src/app/api/stocks/[ticker]/ai-summary/route.ts
import { NextRequest, NextResponse } from "next/server";
import { z } from "zod";
import { getCachedQuote } from "@/lib/data-sources";
import { cached, CACHE_TTL } from "@/lib/redis";
import { buildStockSummaryPrompt } from "@/lib/ai/prompts";
import { generateWithFallback } from "@/lib/ai/summarize";
import { normalizeTicker } from "@/lib/tickers";
import { resolveStock } from "@/lib/stock-service";
import { prisma } from "@/lib/prisma";

const ModelSchema = z.enum(["groq-llama", "gemini-flash", "openrouter-free"]).optional();

export async function GET(
  req: NextRequest,
  { params }: { params: Promise<{ ticker: string }> }
) {
  const { ticker: rawTicker } = await params;
  const ticker = normalizeTicker(rawTicker);
  const model = ModelSchema.parse(req.nextUrl.searchParams.get("model") ?? undefined);

  const { data } = await cached(`ai-summary:${ticker}:${model ?? "default"}`, CACHE_TTL.AI_SUMMARY, async () => {
    const stock = await resolveStock(ticker);
    const { data: quote } = await getCachedQuote(ticker);
    const news = await prisma.newsItem.findMany({
      where: { ticker },
      orderBy: { publishedAt: "desc" },
      take: 3,
    });

    const prompt = buildStockSummaryPrompt({
      ticker,
      name: stock.name,
      price: quote.price,
      weeklyChangePercent: quote.changePercent,
      peRatio: quote.peRatio,
      dividendYield: quote.dividendYield,
      rsi: null,
      headlines: news.map((n) => n.title),
    });

    const result = await generateWithFallback(prompt, model);
    return { summary: result.text, modelUsed: result.modelUsed, modelLabel: result.label };
  });

  return NextResponse.json(data);
}
```

This route accepts an optional `?model=` query param (validated against the `FreeModelId` keys), resolves the stock, gathers quote + recent news headlines + RSI, builds the prompt, calls `generateWithFallback`, and returns `{ summary, modelUsed, modelLabel }`. The result is cached in Redis for 30 minutes keyed by ticker plus model id, since regenerating this on every page view would waste free-tier quota for no benefit.

## Step 7: The frontend — model selector and summary card

Build an `AiSummaryCard` component with: a `Select` dropdown (shadcn) listing the three `FREE_MODELS` options, a "Generate Summary" button, a loading skeleton while the request is in flight, and once loaded, the summary text with a small caption showing which model actually generated it and a colored sentiment badge (green Bullish, red Bearish, gray Neutral).
