# Part 22: UI Polish — Bloomberg Terminal Theme

## Concept

Functionally the app is complete. This part is a focused visual pass to make it *feel* like a professional trading terminal rather than a loose collection of cards: a dark, data-dense theme, consistent typography for numbers, a persistent ticker-search command bar, and a cleaner dashboard layout.

## Step 1: Dark theme as default

We already added `className="dark"` to `<html>` in Part 18's layout. Confirm your `globals.css` (generated by shadcn's `init` in Part 2, Tailwind v4 CSS-first config) has a `.dark` block with slate/zinc-based tokens. Optionally tighten the palette specifically for a terminal feel — adjust the `.dark` CSS variables so background is near-black (`#0a0e14`-ish) rather than shadcn's default dark slate, borders are subtle, and accent colors lean toward amber/cyan (classic terminal accents) for interactive elements, while keeping green/red strictly reserved for gain/loss per our `.text-gain`/`.text-loss` utilities from Part 2.

## Step 2: Monospace numerals everywhere

Terminal-style apps align numbers using tabular/monospace figures so columns of prices line up visually. We already added `.font-mono-num` (using `font-variant-numeric: tabular-nums`) as a Tailwind v4 `@utility` in Part 2. Do a pass across every component built so far (`MetricCard`, quote displays, watchlist table, backtest results, DCA results, REIT metrics) and ensure every numeric value renders with `.font-mono-num` applied consistently.

## Step 3: A global command bar (⌘K-style ticker search)

Build a `CommandBar` component using shadcn's `Command` component (already installed in Part 2) triggered by `Cmd+K`/`Ctrl+K` from anywhere in the app, letting users jump straight to any ticker by typing its symbol or name (reusing the `/api/stocks/search` route from Part 7). Mount it once in the root layout (`src/app/layout.tsx`) alongside a small visible "Search... (⌘K)" button in the header.

```tsx
// src/components/layout/command-bar.tsx
"use client";

import { useEffect, useState } from "react";
import { useRouter } from "next/navigation";
import { CommandDialog, CommandInput, CommandItem, CommandList } from "@/components/ui/command";

export function CommandBar() {
  const [open, setOpen] = useState(false);
  const [results, setResults] = useState<{ ticker: string; name: string }[]>([]);
  const router = useRouter();

  useEffect(() => {
    const onKeyDown = (e: KeyboardEvent) => {
      if ((e.metaKey || e.ctrlKey) && e.key.toLowerCase() === "k") {
        e.preventDefault();
        setOpen((v) => !v);
      }
    };
    window.addEventListener("keydown", onKeyDown);
    return () => window.removeEventListener("keydown", onKeyDown);
  }, []);

  async function search(q: string) {
    if (!q) return setResults([]);
    const res = await fetch(`/api/stocks/search?q=${encodeURIComponent(q)}`);
    const json = await res.json();
    setResults(json.results ?? []);
  }

  return (
    <CommandDialog open={open} onOpenChange={setOpen}>
      <CommandInput placeholder="Search SGX ticker..." onValueChange={search} />
      <CommandList>
        {results.map((r) => (
          <CommandItem
            key={r.ticker}
            onSelect={() => {
              setOpen(false);
              router.push(`/stock/${r.ticker}`);
            }}
          >
            <span className="font-mono-num mr-2">{r.ticker}</span>
            <span>{r.name}</span>
          </CommandItem>
        ))}
      </CommandList>
    </CommandDialog>
  );
}
```

Mount `<CommandBar />` in `src/app/layout.tsx` below the header.

## Step 4: Consistent page layout shell

Create a `src/components/layout/page-shell.tsx` wrapper providing consistent max-width, padding, and a breadcrumb-style header (ticker name + sector badge + "Add to Watchlist" star icon) for all stock-related pages, then refactor the stock detail page (Parts 8-17) to use this shell instead of ad-hoc `<div className="mx-auto max-w-6xl p-6...">` repeated in each page file.

## Step 5: Sticky tab navigation on the stock page

Since the stock detail page now has many tabs (Chart, Indicators, Dividends, DCA Calculator, Backtest, REIT Focus, News), make the `TabsList` sticky beneath the header on scroll (Tailwind's `sticky top-[header-height] z-10 bg-background/95 backdrop-blur` pattern) so users don't need to scroll back up to switch tabs on a long page.

## Step 6: Loading skeleton consistency

Confirm every panel refactored in Part 20 to use `AsyncBoundary` renders skeletons that roughly match the shape of their eventual content (e.g., the chart's skeleton should be a wide rectangle at chart height, not a generic small box) — mismatched skeleton shapes cause visible layout shift and feel unpolished.

## Step 7: Empty and zero-data states

Add explicit, friendly empty-state messaging (not just blank space) for: a ticker with no dividend history ("This stock has no recorded dividend history"), a REIT with unavailable fundamentals data ("REIT fundamentals unavailable — try refreshing"), an empty watchlist ("Your watchlist is empty — search for a ticker to get started"), and a backtest with zero qualifying trades ("No trades matched this strategy in the selected range — try a longer range or different parameters").

## Step 8: Favicon, metadata, and Open Graph

Update `src/app/layout.tsx`'s exported `metadata` object with a proper title template (e.g., `"%s | SGX Dashboard"`), description, and a simple Open Graph image so shared links (e.g., posting your portfolio project on LinkedIn) render nicely with a preview card. In Next.js 16, static `metadata` exports on layouts/pages work the same as prior versions — no async changes needed here (that requirement only applies to `params`/`searchParams`, not `metadata` objects themselves).

## Step 9: A short design-review checklist

Before moving on, review the whole app one more time against this list:
- Do all numbers align and use tabular figures?
- Is green/red used *only* for gain/loss, never decoratively elsewhere (to avoid diluting its meaning)?
- Does every async panel have a matching skeleton, error, and empty state?
- Can you reach any tracked ticker within two actions from anywhere (command bar or nav)?
- Does the app look coherent if you screenshot the home page, a stock page, the REITs page, and the simulator side by side?
