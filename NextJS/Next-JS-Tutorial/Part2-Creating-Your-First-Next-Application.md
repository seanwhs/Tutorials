# Creating Your First Next.js Application

In Part 1, we learned that Next.js is a **full-stack React framework** for building modern web applications.

Now it's time to build your first real Next.js application.

The official command for creating a new project is:

```bash
npx create-next-app@latest
```

Although this appears to be a simple command, it actually performs a tremendous amount of work behind the scenes.

It automatically:

* downloads the latest Next.js starter template
* installs Next.js 16
* installs React 19
* configures TypeScript
* configures ESLint
* configures Tailwind CSS
* sets up the App Router
* enables Turbopack for development
* creates a Git repository
* generates all required configuration files

Think of `create-next-app` as receiving a professionally configured starter application rather than manually assembling dozens of tools yourself.

# Understanding Turbopack

Previous versions of Next.js used **Webpack** as the default development bundler.

Next.js 16 now uses **Turbopack** by default.

Turbopack is:

* written in Rust
* significantly faster than Webpack
* optimized for incremental builds
* capable of near-instant refreshes
* designed specifically for large React applications

This is why modern Next.js applications often start in under a second.

# Common Warning: Workspace Root Detection

Some developers encounter a warning like:

```text
Warning: Next.js inferred your workspace root,
but it may not be correct.
```

For example:

```text
Detected multiple lockfiles:

Documents/package-lock.json
learn-next/package-lock.json
```

This happens because Next.js searches upward through parent folders looking for workspace boundaries.

Example:

```text
Documents/
в”њв”Ђв”Ђ package-lock.json
в””в”Ђв”Ђ learn-next/
    в””в”Ђв”Ђ package-lock.json
```

---

## Recommended Fix

Delete the accidental parent lockfile.

Windows:

```powershell
Remove-Item ..\package-lock.json
```

Linux/macOS:

```bash
rm ../package-lock.json
```

---

## Alternative Fix

You can manually configure the Turbopack workspace root:

```ts
import path from "path";
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  turbopack: {
    root: path.resolve(__dirname),
  },
};

export default nextConfig;
```

Most developers never need to do this.

# Important Warning: Avoid `npm audit fix --force`

After installation, you may run:

```bash
npm audit
```

and see vulnerability warnings.

Do **not** immediately run:

```bash
npm audit fix --force
```

This command may downgrade important framework dependencies.

For example:

```text
next@16
в†“
next@9
```

This happens because npm prioritizes dependency constraints over framework compatibility.

Instead, use:

```bash
npm audit
npm audit fix
```

Or simply ignore framework advisories in a freshly generated project.

> **Rule of thumb:** Never run `npm audit fix --force` immediately after creating a modern framework application.

# Understanding the Generated Files

## `package.json`

This is the heart of your application.

It contains:

* project metadata
* installed packages
* npm scripts
* dependency versions

Example:

```json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start"
  }
}
```

Common commands:

```bash
npm run dev
```

Starts the development server.

```bash
npm run build
```

Creates a production build.

```bash
npm start
```

Runs the production build locally.

---

## `next.config.ts`

This file configures Next.js itself.

A newly created project typically contains:

```ts
import type { NextConfig } from "next";

const nextConfig: NextConfig = {};

export default nextConfig;
```

Later, you'll use this file for:

* image optimization
* caching
* middleware
* Turbopack settings
* experimental features
* build configuration

---

## `tsconfig.json`

This configures TypeScript behavior.

Example:

```json
{
  "compilerOptions": {
    "target": "ES2017"
  }
}
```

Even if you don't know TypeScript yet, leave this file alone for now.

---

## `next-env.d.ts`

This file allows TypeScript to understand Next.js-specific features.

You should never edit this file manually.

---

## `eslint.config.mjs`

This configures ESLint.

ESLint helps detect:

* bugs
* syntax errors
* code style problems
* React mistakes

---

## `postcss.config.mjs`

This configures PostCSS, which Tailwind CSS uses internally.

Most projects never modify this file.

---

## `README.md`

This file contains project documentation.

As your project grows, this file becomes increasingly important.

---

## `AGENTS.md`

Newer versions of `create-next-app` may generate an `AGENTS.md` file.

This file is intended for AI coding agents and tools, documenting project conventions and instructions.

As AI-assisted development becomes more common, files like this will become increasingly important.

---

## `CLAUDE.md`

Some development environments or AI tooling setups create a `CLAUDE.md` file.

This file typically contains instructions or context for AI assistants working with your repository.

This file is optional and not part of core Next.js.

# File-System Routing: The Magic of Next.js

The App Router uses your folder structure as your URL structure.

For example:

```text

в”њв”Ђв”Ђ page.tsx
в”њв”Ђв”Ђ about/
в”‚   в””в”Ђв”Ђ page.tsx
в””в”Ђв”Ђ contact/
    в””в”Ђв”Ђ page.tsx
```

automatically creates:

```text

/about
/contact
```

No route configuration is required.

No router setup is required.

No route table is required.

Folders become URLs.

# What Does `export default` Mean?

Every page file must export a default component:

```tsx
export default function Page() {
  return <div>Hello</div>;
}
```

The function name itself is not important.

The `default export` is what Next.js uses to render the page.

---

# JSX Refresher

This syntax:

```tsx
<h1>Hello World</h1>
```

is called **JSX**.

JSX allows you to write HTML-like syntax inside JavaScript and TypeScript.

Internally:

```tsx
<h1>Hello</h1>
```

becomes something conceptually similar to:

```js
React.createElement(
  "h1",
  null,
  "Hello"
);
```

Modern React performs this transformation automatically.
